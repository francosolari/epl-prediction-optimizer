## Design Context

### Users
The app is used locally by one technical user managing a Premier League weekly
selection challenge. The user needs to refresh data, review model
recommendations, compare close alternatives, record the actual submitted pick,
and preserve notes about human overrides such as injuries or lineup news.

### Brand Personality
Quiet, analytical, decisive. The UI should feel like a compact football trading
desk: dense but readable, fast to scan, and built for repeated weekly use.

### Aesthetic Direction
Light-mode operational dashboard with restrained matchday color, sharp tables,
compact controls, and no heavy frontend framework. The design should avoid
marketing-page decoration, generic card grids, React-style app chrome, or visual
noise that slows down weekly decision-making.

### Design Principles
- Recommendations are advice; actual submitted picks are the source of truth.
- Show constraints at the point of decision: team use, venue use, result status.
- Make close calls visible so human context can override the model deliberately.
- Keep every workflow one local HTML form away; no React or build pipeline.
- Prefer dense, legible match rows over decorative panels.
