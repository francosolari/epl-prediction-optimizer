1. State of the Art — What the Best Models Actually Do

Core finding No single model beats the bookmaking market consistently over large samples. The goal is not to be "smarter" than odds — it's to make better constrained decisions given our season-long pick structure. That's a fundamentally different optimisation target that the market does not model.

Accuracy benchmarks (pre-match, three-outcome)

ApproachAccuracyNotes
Random baseline~33%Three equal outcomes
Elo only52–55%Good signal, lacks calibration
Bookmaker consensus odds55–58%Best freely available pre-match signal
XGBoost + engineered features56–67%High variance by study
FiveThirtyEight SPI~57–60%Shot-based; outperforms Elo
Our current model (GBM + decay weights)54–58%Competitive for a goals-only model
Neural networks (best reported)61–65%Requires event-level data (Opta/StatsBomb)
Key insight Accuracy improvements above ~58% pre-match require shot-level or event-level data (xG, pass networks, pressing metrics). The next 2–4 percentage points from our current position are achievable with data we already have or can easily obtain.

Our football-data.co.uk CSVs contain far more than we parse. Every completed-season CSV from 2000–01 onwards has:

High impact · In-hand

Shots on Target (HST / AST)

Best cheap proxy for xG. Rolling 5-game HST average correlates strongly with future goal-scoring. Available from 2000–01.

High impact · In-hand

Betting Odds (B365H/D/A, AvgH/D/A)

Market implied probabilities aggregate information (injuries, team news) we can't model. Available from 2002–03. Most powerful feature we're not using.

Medium impact · In-hand

Corners (HC / AC)

Attacking pressure proxy. Rolling corners correlate with possession and chance creation. Available from 2000–01.

Medium impact · In-hand

Half-time Scores (HTHG / HTAG)

Not useful pre-match, but rolling HT performance (half-time points) indicates fast-start tendencies — useful form signal.

Medium impact · In-hand

Yellow / Red Cards (HY, AY, HR, AR)

Disciplinary tendencies affect match dynamics. Teams with high card rates tend to have different home/away draw profiles.

Requires fetch · Free

understat.com xG Data

Per-match xG for/against, available from 2014–15 onwards. The single biggest potential accuracy jump with new data.

Important Shots, corners, and cards are post-match statistics — we can only use their rolling historical averages as pre-match features. Using the current match's stats would be leakage. Our feature pipeline already handles this correctly for goals; the same pattern applies.

Key learnings

  • Isotonic calibration hurts vs sigmoid — needs more data, and our 11k-row training set isn't large enough for it.
  • Exponential time-decay weights (decay=0.9/season) consistently improve 2425 performance, confirming the recency hypothesis.
  • Pick variance is high: 33–35 picks means even a few different fixture selections from changed probabilities can swing ±10 points. Both seasons together is the right optimisation target.
  • Expected points are stable (~63–66) across configs — the model's overall pick quality is similar. Actual points variation above that is match-result luck.
  • Elo is still the dominant feature (10–20× permutation importance of any other feature). Don't remove it.

4. Prioritised Improvement Roadmap

1

Rolling shots on target (HST/AST) as features

Already in our raw CSVs. Parse HST/AST per match, build rolling 5-game averages as pre-match features. This is a cheap xG proxy that covers all 25 seasons with shot data. Expect +1–3% accuracy. Implementation: update data/sources.py to extract these columns, add to feature pipeline.

2

Betting market implied probabilities as features

B365H/D/A and AvgH/D/A odds are already in our CSVs from 2002–03. Convert to implied probabilities (1/odds, normalised). Market odds encode team news, injuries, and tactical info we can't model. This is the single biggest expected improvement. Caveat: introduces dependency on historical odds availability.

3

Dixon-Coles draw-probability correction

Post-process Stage A output: apply a small rho adjustment (−0.03 to −0.1) to slightly increase P(draw) for matches where both teams have similar expected goals. Specifically helps with 0-0 and 1-1 under-prediction. Can be implemented as a calibration layer without retraining the GBM.

4

understat.com xG data (2014–15 onwards)

Free API, per-match xGF/xGA available for all EPL teams. Would give us true shot quality signal for ~10 seasons. Rolling xG differential outperforms rolling goal differential for future performance prediction. Requires a new data source module and a join with historical matches on date/team.

Stage B optimiser improvements The ILP optimiser already does near-optimal pick selection given Stage A probabilities. The biggest Stage B gains come from better Stage A calibration (so the probability gaps between picks are more reliable), not from changing the optimisation formulation. One exception: consider a "regret-minimisation" objective variant for weeks where multiple fixtures have similar expected points — currently the ILP is indifferent to ties.

5. FiveThirtyEight SPI Deep-Dive

FiveThirtyEight's Soccer Power Index (SPI) is the most publicly documented professional football prediction system. It goes substantially beyond Elo:

Core components

  • Adjusted Goals: reduces value of goals scored during player-advantage situations (red cards) and late in blowouts — prevents garbage-time goals from inflating ratings.
  • Shot-Based xG: estimates expected goals from each shot using distance, angle, body part, and individual player conversion history. This is the core signal.
  • Non-Shot xG: expected goals from non-shooting actions — passes, interceptions, and take-ons in dangerous areas. Captures possession quality beyond shots.
  • Match Importance: down-weights matches where one team is in garbage time (promotion/relegation already secured). Prevents noise from low-motivation games polluting ratings.

Model architecture

  • Projects home and away goals as independent Poisson distributions.
  • Runs 20,000 Monte Carlo season simulations per week — not just match-by-match prediction.
  • Uses 550,000+ matches back to 1888, with time-decay weighting (similar to what we implemented).
  • Opta play-by-play data from 2010 onwards is the backbone — not publicly available.
What we can and cannot replicate Shot-based xG requires event-level data (Opta, StatsBomb, or understat). Non-shot xG requires pass-tracking data. The Adjusted Goals concept is approximable by discounting goals from matches with red cards — this is achievable. Match Importance could be approximated with our in-season table position features (already implemented).

6. Dixon-Coles & Draw Prediction

The Dixon-Coles model (1997) is the statistical foundation of most professional football prediction. Its key insight: standard Poisson models systematically underestimate draws and low-scoring results (0-0, 1-0, 0-1, 1-1).

The rho correction

For score outcomes (i, j) where i ≤ 1 and j ≤ 1, Dixon-Coles applies a correction factor τ:

τ(i=0, j=0) = 1 - λ_home × λ_away × ρ
τ(i=1, j=0) = 1 + λ_away × ρ
τ(i=0, j=1) = 1 + λ_home × ρ
τ(i=1, j=1) = 1 - ρ
# All other scores: τ = 1 (unchanged)
# ρ typically estimated as −0.03 to −0.15 (negative → boosts 0-0 and 1-1 probability)

Time decay

Dixon-Coles originally proposed exponential time-weighting of matches — exactly what we implemented in May 2026. Optimal decay found empirically: ξ ≈ 0.00325 per day, or roughly 0.88–0.92 per season — consistent with our 0.9/season.

Draw prediction — the hardest problem

Draws are systematically under-predicted by all standard approaches. Specific findings from literature:

  • A 2025 study achieved 86.7% overall accuracy but substantially lower draw precision.
  • The Soccer Prediction Challenge (2023) explicitly flagged draw prediction as unsolved.
  • Best mitigation: Dixon-Coles rho correction + draw-tendency rolling feature (we have the latter).
  • Class weighting (upweight draw examples during training) helps recall but hurts precision — use carefully.
  • Draw rate varies by team pair — some fixtures are historically draw-heavy (we have H2H draw rate as a feature).
Our draw features (already implemented) We have home_draw_tendency, away_draw_tendency (rolling 10-game draw rates per team) and h2h_draw_rate (last 5 meetings between the specific pair). These are the right features. The next step is the Dixon-Coles post-processing layer on top of our GBM output.

7. xG — When It Helps & When It Doesn't

xG (expected goals) is often cited as transformative, but the empirical picture is nuanced:

ContextBest metricAccuracyWhy
Post-match (explain result) xG65.6% Shots are the only way to score; shot quality reflects actual in-game dominance
Pre-match (predict result) EPV (expected possession value)58.3% Possession quality predicts future shot creation; xG without context is weaker pre-match
Pre-match (rolling xG average) xG for/against 5-game average~56–58% Historical xG is a better quality signal than historical goals (less noisy)
No xG available Shots on target rolling average~55–57% HST is a 0.85-correlation proxy for xG, freely available

In 38.7% of EPL matches, the team with higher cumulative xG does NOT win — confirming xG improves calibration but cannot eliminate variance. For our use case (picking one winner per week), this variance matters enormously: a 65% win-probability pick still loses 35% of the time.

Practical recommendation

Use rolling shots on target (HST/AST) as the first step — available immediately from our existing CSVs. Fetch understat xG data as a second step for the seasons where it's available (2014–15+), and use it to replace or augment the HST feature for recent seasons.

8. Betting Odds as Features

Using bookmaker odds as model inputs is controversial but consistently effective in academic literature. The key argument: odds incorporate information we cannot model (injuries confirmed hours before kickoff, tactical surprises, referee assignments, travel fatigue).

Available in our CSVs

  • B365H / B365D / B365A: Bet365 opening odds (home win / draw / away win). Available from 2002–03.
  • AvgH / AvgD / AvgA: Market average opening odds across bookmakers.
  • MaxH / MaxD / MaxA: Best available odds (highest market efficiency signal).
  • B365CH / B365CD / B365CA: Bet365 closing odds — closest to kickoff, highest signal quality.

How to use them correctly

# Convert odds to implied probability (removes overround)
p_home_raw = 1 / B365H
p_draw_raw = 1 / B365D
p_away_raw = 1 / B365A
overround = p_home_raw + p_draw_raw + p_away_raw
p_home = p_home_raw / overround   # normalised implied probability
p_draw = p_draw_raw / overround
p_away = p_away_raw / overround
Philosophical consideration Using odds as features means our model is partly a "market refinement" — we're learning where our other features agree or disagree with the market. The value comes from combining odds with our structural features (Elo, form, H2H) which the market also uses but may weight differently for our specific optimisation task (season-long constraint picks). In practice: odds + Elo + form consistently outperforms any single signal alone.

Backtesting caution

When backtesting with odds features, ensure you use opening odds (available before the match) not closing odds (which incorporate in-day information). Our CSVs have both — use B365H/D/A, not B365CH/D/A, for pre-match features.

Frequently cited

xG proxy, consistently 3rd–5th most important feature in shot-data studies. We can add this now.

Frequently cited

Market odds (implied probs)

Outperforms most individual features in isolation. Best used alongside structural features.

Niche / High data cost

Player quality (FIFA ratings)

2023 PLOS ONE study got F1=0.47 using FIFA position ratings. Requires squad-level data per season.

Model architecture findings

  • XGBoost / GBM consistently top-performing for tabular match data — our choice is correct.
  • Neural networks only win when event-level data is available (Opta play-by-play). On tabular features, GBM is equal or better.
  • Ensemble / stacking: combining GBM output with a simple Poisson model modestly improves calibration. Diminishing returns vs complexity.
  • Calibration matters more than accuracy for our use case — Stage B optimizer is sensitive to probability ordering and magnitude, not just which outcome is most likely.
  • Feature engineering > model complexity for 11k-row datasets. More data or better features beat deeper architectures at this scale.
